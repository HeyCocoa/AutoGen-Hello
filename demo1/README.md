# 选题策略生成器 (Topic Strategy Generator)

这是一个多智能体协作的选题策略生成器：先把业务场景里最关键的缺口补齐，再用搜索和分析把方向跑顺，最后把结论、依据和争议点整理成一份能直接拿去用的策略文档。

## 角色分工

系统包含 4 个智能体角色，采用顺序协作模式：

1. **Clarifier（澄清者）**
   - 先扫一遍输入，抓住真正影响策略的空白
   - 根据业务类型调整提问角度，不硬套模板
   - 问得少但准，让用户能很快补齐继续往下走

2. **Analyst（分析师）**
   - 先交一份搜索大纲，保证检索方向不跑偏
   - 用真实搜索结果支撑结论，而不是凭感觉说话
   - 给出清晰的策略方向和优先级，便于直接执行

3. **Critic（质检员）**
   - 先看搜索大纲有没有漏掉关键维度
   - 挑战分析结论，专门找逻辑漏洞和证据短板
   - 该补的补、该改的改，把风险点说清楚

4. **Writer（撰写者）**
   - 把分析和质检合到一起，不是简单复制粘贴
   - 该保留的依据和争议点都留着，读者好判断
   - 输出一份结构清楚、拿来就能用的策略文档

## 工作流与设计理由（每点一句）

1. **澄清阶段**：先补齐关键信息，避免后续分析走偏或输出空话。  
2. **搜索大纲对齐**：分析师先交“搜索大纲”，质检员拦截缺漏，保证搜索方向覆盖核心痛点。  
3. **分析阶段**：基于真实搜索结果给出判断，降低拍脑袋决策的风险。  
4. **质检阶段**：从反方视角拆逻辑与补盲点，避免单一视角造成误判。  
5. **文档阶段**：将证据链、结论与争议点一起落到结构化文档，便于复用与复核。
